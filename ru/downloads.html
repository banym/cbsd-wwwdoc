Все .img (USB Memstick образы) могут быть записаны на USB носитель через команду:

	    % dd if=_path_to_img.img_ of=/dev/da0 bs="10240" conv="sync"

Будьте внимательны, прописывая корректное имя устройства в of= аргументе.

В данный момент, генерация CBSD образов отключена. Для установки из исходников воспользуйтесь этой инструкцией

Если же все равно что-то непременно хотите скачать ;-), скачайте FreeBSD LiveCD из rescue jail: Rescuecbsd LiveCD

Список преднастроенных FreeBSD Jail-based имиджей

Ниже приведен список преднастроенных имиджей, доступные для скачивания через команду cbsd repo action=get sources=img name=name.

Данные образы могут представлять из себя «кубики» из которых можно собирать one-click необходимую вам инфраструктуру. Вы ограничены только физическими ресурсами сервера, по исчерпанию которых всегда можно поставить еще одну ноду и мигрировать часть клеток на нее.

Каждый образ представляет из себя архив клетки с необходимым ПО и имеющим минимальные шаблоны конфигурационных файлов, на которые отрабатывает хелпер применяющий ввод первоначальных пользовательских данных (например пароль администратора приложения, имя сайта и тд.) для того, чтобы приложение заработало как можно быстрее. Хелперы могут конфигурировать как stand-alone клетку (база, WEB сервер и приложение - все в одном) либо, если клетка это позволяет, использовать внешние базы данных и WEB сервера для организации, например, VHOST-инга. После инсталляции, работать с клеткой, сопровождать конфигурацию и ПО можно стандартным способом — она независима от CBSD. Все хелперы могут отрабатывать неинтерактивно (все возможные аргументы доступны по команде --help), являясь примером массового построения и разворачивания Application серверов из клеток в автоматическом режиме.

Хелперы протестированы на FreeBSD ОС, на которой ведется разработка, тем не менее, они способны создавать те же конфигурации в Linux системах, которая пока для меня не является приоритетной.
