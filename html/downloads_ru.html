Все .img (USB Memstick образы) могут быть записаны на USB носитель через команду: 
dd if=_path_to_img.img_ of=/dev/da0 bs="10240" conv="sync"
Будьте внимательны, прописывая корректное имя устройства в of= аргументе.

В данный момент, генерация cbsd образов отключена. Для установки из исходников воспользуйтесь этой инструкцией
Если же все равно что-то непременно хотите скачать ;-), скачайте FreeBSD LiveCD из rescue jail: Rescuecbsd LiveCD

Если вы хотите использовать pkg репозитории из которых собраны клетки, создайте файл /usr/local/etc/pkg/repos/freecbsd.conf с таким содержимим, подставив в качестве url одну из перечисленных ниже ссылок: 
  freecbsd: {
    url: "URL_HERE",
    mirror_type: "srv",
    enabled: yes
  }
где URL_HERE:

FreeBSD 9.2
pkg+http://dl.bsdstore.ru/freebsd/amd64/9.2/xorg # (обычная версия, NVIDIA GL)
pkg+http://dl.bsdstore.ru/freebsd/amd64/9.2/xorgkms # (Xorg with KMS)
pkg+http://dl.bsdstore.ru/freebsd/amd64/9.2/nox # (WITHOUT_X11=yes, php55, MariaDB)
FreeBSD CURRENT aka 10.0
pkg+http://dl.bsdstore.ru/freebsd/amd64/10.0/xorg # (обычная версия, NVIDIA GL)
pkg+http://dl.bsdstore.ru/freebsd/amd64/10.0/xorgkms # (Xorg with KMS)
pkg+http://dl.bsdstore.ru/freebsd/amd64/10.0/nox # (WITHOUT_X11=yes, php55, MariaDB)

Прим1: в данный момент сборка выполняется только для x86-64 архитектуры пр причине отсутствия в моей ферме иных.
Прим2: данные имиджи и репозитории подняты и поддерживаются больше в целях демонстрации и проверок функционала cbsd, нежели для обеспечения чьей-то работы. В целях безопасности, приучайте себя не доверять любым репозиторям ПО в интернете, используйте только официальные и хорошо проверенные репозитории либо собирайте собственные.

Ниже приведен список преднастроенных имиджей, доступные для скачивания через команду cbsd repo action=get sources=img name=name. Данные образы могут представлять из себя "кубики" из которых можно собирать необходимую вам инфраструктуру. Вы ограничены только физическими ресурсами сервера, по исчерпанию которых всегда можно поставить еще одну ноду и мигрировать часть клеток на нее.
Каждый образ представляет из себя архив клетки с необходимым ПО и имеющим минимальные шаблоны конфигурационных файлов, на которые отрабатывает хелпер применяющий ввод первоначальных пользовательских данных (например пароль администратора приложения) для того, чтобы приложение заработало как можно быстрее. Далее работать с клеткой, сопровождать конфигурацию и ПО можно стандартным способом - она независима от cbsd. Все хелперы могут отрабатывать неинтерактивно (все возможные аргументы доступны по команде --help), являясь примером массового построения и разворачивания Application серверов из клеток в автоматическом режиме.
